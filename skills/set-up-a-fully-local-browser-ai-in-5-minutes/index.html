Set Up a Fully Local Browser AI in 5 Minutes (No Cloud, No API Key)

Source ↗ Tools & Platforms

What This Skill Is

You can run a fully private, on-device AI agent inside Chrome — no API keys, no cloud, no data leaving your laptop. Nico Martin's Chrome extension loads a 2B-parameter Gemma 4 model (~500MB) into your browser via WebGPU. It can read open tabs, search your browsing history semantically, summarize the current page, and highlight DOM elements — all locally.

This is the Transformers.js + Gemma 4 + WebGPU stack: a real production-ready architecture for shipping privacy-respecting AI features inside any browser extension.

Why It Works

The extension uses semantic embeddings on your pages — it generates embeddings of headings, paragraphs, and lists, then returns only the most relevant chunks for your question. The model never sees the full DOM. Speed is ~5-10 tokens/sec on integrated GPU (faster on dedicated cards). Background service worker loads the model once and persists it across tabs, so you pay the loading cost only once per session.

How to Use It

  1. Install the extension from the Chrome Web Store — one click
  2. First load triggers a one-time download of Gemma 4 E2B ONNX model and all-MiniLM-L6-v2 embedding model (~500MB total, cached after that)
  3. Open the side panel and chat

For more agent-like control (click/type/scroll DOM, screenshot capture), use Gemma Gem instead — it adds a run_javascript tool and lets you choose between Gemma 4 E2B (~500MB) or E4B (~1.5GB).

Tools available on-device: get_open_tabs, ask_website (semantic search on current page), find_history (semantic search across browsing history), summarize_tab, highlight_element

Prompt Template

You have access to my open tabs and the current page.

Find the tab where I was reading about [topic], summarize what it says about [specific question], and tell me whether the article disagrees with what I'm currently reading.
