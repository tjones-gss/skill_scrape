How to Keep Your AI Coding Agent From Nuking Production

Source ↗ Coding

What This Skill Is

A 4-layer security setup for running coding agents (Cursor, Claude Code, Codex CLI, etc.) that prevents a 9-second production wipe. The real fix isn't being more careful with prompts — it's configuring the environment so "be careful" is the default, not a prayer.

Based on the April 2026 Cursor/Claude database deletion incident, Anthropic's official dev container guide, and Trail of Bits' hardened sandbox.

Why It Works

Same four layers apply to any coding agent: Cursor, Codex CLI, Aider, etc. The cleanup from a production wipe took days. The agent took 9 seconds. This setup ensures the agent's 9 seconds of speed never costs you days of cleanup.

How to Use It

  1. Worktree: always run agents in a separate git worktree, never on main
  2. Container: use Anthropic's official claude-code Dev Container Feature — container sees only the bind-mounted project folder
  3. Deny-list: commit .claude/hooks/pre_tool_use.sh so every developer inherits the same baseline
  4. Permissions: never use --dangerously-skip-permissions on the host; only inside the container

For the paranoid, use Trail of Bits' hardened devcontainer: adds network egress allowlists (only api.anthropic.com, github.com, npmjs.org), drops all Linux capabilities, and forbids privilege escalation.

Prompt Template

# Coding Agent Safety Setup

## Layer 1 — Isolated git worktree (never work on main):
git worktree add ../myproject-agent-task feature/agent-branch
cd ../myproject-agent-task
claude --worktree

## Layer 2 — Sandboxed dev container (.devcontainer/devcontainer.json):
{
  "name": "Claude Code Sandbox",
  "image": "mcr.microsoft.com/devcontainers/base:bookworm",
  "features": {
    "ghcr.io/anthropics/devcontainer-features/claude-code:1.0": {}
  },
  "remoteUser": "vscode"
}

## Layer 3 — Command deny-list hook (.claude/hooks/pre_tool_use.sh):
#!/bin/bash
command=$(jq -r '.tool_input.command' < /dev/stdin)
case "$command" in
  rm\ -rf*|*"DROP TABLE"*|*"DELETE FROM"*|*"git push --force"*|*"truncate"*)
    echo "Blocked: destructive command requires manual run" >&2
    exit 2
    ;;
esac
exit 0

## Layer 4 — Require explicit confirmation for destructive operations:
Use auto mode (not --dangerously-skip-permissions).
Set permissions.disableBypassPermissionsMode: "disable" in managed settings.
