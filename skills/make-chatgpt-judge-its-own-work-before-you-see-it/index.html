Make ChatGPT Judge Its Own Work Before You See It

Source ↗ June 21, 2026 Coding

What This Skill Is

Use an LLM-as-a-judge loop with Codex's /goal command. Instead of giving Codex a task and hoping for the best, you first ask the model to define what "good enough" looks like — then it works in a loop checking its own output against those criteria before reporting back to you.

Why It Works

Most AI failures happen because there's no clear success criteria. When you ask Codex to define and then enforce its own acceptance criteria, it self-corrects rather than stopping at "looks done." The loop continues until the work measurably meets the goal — not just when the model runs out of ideas.

How to Use It

Use in Codex or Claude Code with any task that has a measurable outcome: dashboard performance, document cleanup, bug fixes, code optimization. Works especially well with /goal prefix.

Prompt Template

/goal

Work toward this outcome:
[describe the task]

Before starting, define what "good enough" means for this task.

Create 3-5 success criteria Codex can keep checking while it works. Include any hard requirements, tests, files, constraints, style rules, performance targets, or user-facing behavior that must be preserved.

Then work in a loop:
1. Make the next useful improvement.
2. Judge the result against the success criteria.
3. Identify the biggest remaining gap.
4. Continue until the work meets the goal.

Return:
- The final result
- The success criteria you used
- The biggest change you made
