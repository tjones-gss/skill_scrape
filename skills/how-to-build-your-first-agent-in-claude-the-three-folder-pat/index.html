How to Build Your First Agent in Claude: The Three-Folder Pattern That Actually Scales

Source ↗ Agents

What This Skill Is

"How do I start building agents in Claude?" — the most-requested skill in The Neuron's April reader inbox. Jeff Su's canonical guide boils down to three CLAUDE.md files that stack:

  1. Root CLAUDE.md (under 300 lines): global voice rules and model defaults (Sonnet 80% of the time; Opus when a task has 3+ dependent steps)
  2. Workstation CLAUDE.md files: one per area of life (Email HQ, Newsletter HQ, Personal Finances) — rules stack on top of root automatically
  3. Project subfolders: one-off work like a mortgage refinance or Q4 trip — each gets its own CLAUDE.md and MEMORY.md

Every folder also gets a MEMORY.md that Cowork writes to whenever you correct it. Corrections compound.

Why It Works

The three-folder hierarchy separates global preferences from domain-specific rules from project-specific context. Each layer inherits from the one above, so corrections and rules compound over time without contradicting each other.

How to Use It

  1. Open Claude Cowork (desktop app) — for non-developers, this is the right starting point
  2. Paste the prompt above into a fresh chat
  3. Let Cowork create the folder structure and analyze your email patterns
  4. Connect Gmail, Drive, and Calendar via Connectors
  5. Type /schedule to make any task recurring

Prompt Template

Create a new workstation called "Email HQ" in /Documents/Cowork OS.
Inside it, create:
1. CLAUDE.md with email-specific rules (default greeting, sign-off, formality level).
2. An empty MEMORY.md (you'll add learnings to this over time).
3. A "Resources" subfolder.
Then use the Gmail connector to search my last 4 weeks of sent emails. Extract the patterns I use repeatedly: my opener, my sign-off, my formality level, common phrases I use with specific people. Save these as Editorial Rules inside CLAUDE.md.
When you're done, write one paragraph summarizing what you learned about my email voice so I can verify before we lock it in.
