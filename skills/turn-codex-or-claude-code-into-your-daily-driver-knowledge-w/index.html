Turn Codex (or Claude Code) Into Your Daily-Driver Knowledge Work Cockpit

Source ↗ Coding

What This Skill Is

You probably use Codex or Claude Code for code. But Austin Tedesco (head of growth at Every) spends roughly 80% of his working day inside Codex doing go-to-market plans, recruiting outreach, KPI tracking, and emails.

The trick: treat the desktop app (not the CLI) as an "agent management interface" for everything you do.

Why It Works

The desktop app's persistent folder structure means each work area keeps its own context. Connect the right tools, add a project brief, and the agent can do knowledge work across your whole stack — not just code.

How to Use It

  1. Make a folder per work area (e.g., "growth OS") — folders give you persistent named chats per project
  2. Connect plug-ins for everything you live in: Gmail, Slack, Notion, Stripe, data sources
  3. Drop in a markdown project file that explains your business, goals, and how you like to work
  4. Add reviewer agents — fork for strategic alignment and data accuracy checks
  5. Run the starter prompt above in a fresh chat inside that folder
  6. Always do the final human review in the external app (Slack drafts, Gmail drafts), never inside Codex

Prompt Template

Go take a look at the things I use the most (@Notion, @Slack, and @Gmail) and think of some automations that would help me with my work. For each one, explain what it does, when it should run, and which tool it touches. Ask me which ones look good before you build any of them.
