Give Your AI Agent a "Trust Battery" Before Handing It Real Autonomy

Source ↗ Agents

What This Skill Is

Every new AI agent starts at 20% trust. The battery charges with clean execution; it drains every time you repeat yourself. Only hand the agent tasks that match its current charge:

  • 20-40% (Pair mode): You review every output — for money, clients, or permanent side effects
  • 40-60% (Async with checkpoints): Agent works alone, stops at key decisions
  • 60-80% (Full delegation with audit): Agent runs end-to-end; you spot-check the next morning
  • 80%+ (Autonomous): Agent decides when to ask

Why It Works

Most people give agents too much autonomy too fast, then abandon them when they fail. The trust battery framework makes autonomy a function of demonstrated reliability — the same way you'd onboard a new human employee.

How to Use It

  1. Drop this into a Claude Routine scheduled for midnight
  2. Every morning you'll see whether your agent earned more autonomy overnight
  3. Paste the JSON into your ops doc to track trust over time
  4. The self-updating CLAUDE.md loop means the agent keeps improving without you rewriting prompts

Prompt Template

Review today's task outputs. For each one:
(1) Rate execution quality 1-5
(2) Flag any repeated corrections from me
(3) Propose one update to your CLAUDE.md or system prompt that would prevent the correction next time
(4) Decide whether your trust battery should go up, down, or flat

Output a single JSON summary I can paste into our ops doc tomorrow.
