The Opus 4.7 + GPT-5.5 Handoff

Source ↗ Coding

What This Skill Is

Dan Shipper found the cheat code for senior-engineer-level coding: use Opus 4.7 to write the plan, then hand that plan to GPT-5.5 to execute.

On Every's internal Senior Engineer benchmark:

  • Human senior engineers: 80-90
  • This combo: 62.5/100
  • Opus 4.7 alone: low 30s
  • GPT-5.5 alone: low-to-mid 40s

Why it works: Opus 4.7 writes tight, contract-style plans (exact file counts, line limits, first-principles design). That terseness is what gives GPT-5.5 the confidence to delete files, rewrite from scratch, and carry a multi-hour refactor through to completion.

Why It Works

Opus 4.7 is a strong planner; GPT-5.5 is a bold executor. Separating planning from execution — and matching each task to the right model — produces better results than either model doing both.

How to Use It

  1. Open Claude with Opus 4.7 selected
  2. Ask it to write a rewrite plan for your target codebase
  3. Paste that plan into Codex or ChatGPT with GPT-5.5 selected
  4. Use the prompt above to execute
  5. Tip: if you only have GPT-5.5, add explicit "here's what good looks like" targets (e.g., "this file should be ~100 lines when done") to draw out its boldness

Prompt Template

Here is a plan written by a senior engineer for rewriting this codebase from first principles. Execute it faithfully. Do not patch around the existing code: delete what the plan says to delete, rewrite what it says to rewrite, and match its conceptual structure exactly. Carry the plan through from start to finish.
