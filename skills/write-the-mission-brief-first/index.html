Write the Mission Brief First

Source ↗ Coding

What This Skill Is

Want Codex to keep working instead of stopping after one polite little answer?

Try this trick from @meta_alchemist: ask Codex to write the /goal prompt first. /goal is an experimental Codex CLI feature that helps Codex pursue long-running work toward a clear stopping condition.

> read this session and repo, analyze deeply the exact intent and goals we are looking to achieve here then write me the /goal prompt for this.

Why It Works

Agents lose context over long tasks. Making the agent define its own goal first forces it to deeply analyze the intent before acting — and gives you a checkpoint to verify you're aligned before it runs.

How to Use It

  1. Open Codex with your session and repo context loaded
  2. Ask it to write the /goal prompt before it starts working
  3. Paste the resulting /goal back in at the start of your task
  4. Outside Codex: use the same idea with ChatGPT or Claude — have it read your context and write the system prompt before it starts your memo, strategy doc, or meeting follow-up

Prompt Template

read this session and repo, analyze deeply the exact intent and goals we are looking to achieve here then write me the /goal prompt for this.

make sure to dig into history & docs we have to be 100% clear
if you are not sure about certain parts or wanna ask me a few questions to clarify certain goals further don't hesitate
Then paste Codex's answer back in with /goal at the front.
