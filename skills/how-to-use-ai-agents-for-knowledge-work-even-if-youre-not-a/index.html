How to Use AI Agents for Knowledge Work (Even If You're Not a Coder)

Source ↗ Coding

What This Skill Is

Instead of asking your AI to answer a question, ask it to build a tool that solves a recurring problem. The shift is from "help me with this task" to "build me something that handles this task forever."

Greg Brockman described OpenAI employees who aren't engineers using Codex to automate their work — the communications team hooks it to Slack and email, synthesizes feedback, builds internal tools. No coding required.

Why It Works

Computers were always supposed to contort to the human, not the other way around. The moment you start describing your problems instead of phrasing perfect questions, AI becomes dramatically more useful — because specific pain points give the AI enough context to build something real.

How to Use It

  1. Identify a task you do repeatedly (weekly reports, inbox triage, data formatting)
  2. Describe it specifically — not "help me with email" but "every Monday I pull metrics from three dashboards, copy them to a spreadsheet, and format a summary for my manager"
  3. Paste the prompt above with your specific task filled in
  4. Follow the step-by-step setup Claude provides

Prompt Template

I have a recurring task at work: [describe the task precisely].

I currently do it [how often] and it takes about [time].

Build me a simple tool or workflow that automates this.
Walk me through setup step by step, assuming I have zero coding experience.
If it requires any tools or accounts, tell me which ones and how to set them up.
